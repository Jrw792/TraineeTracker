# Training Program — Static Site

A read-only, mobile-friendly site for trainers to look at the active roster, individual trainee summaries, and archived session notes. Designed to be hosted on GitHub Pages.

The site is plain HTML/CSS/JS with no build step required to run — `build.js` only regenerates the session index after you add a new note.

## Layout

```
site/
├── index.html          ← app shell
├── styles.css          ← futuristic-tech theme (light + dark)
├── app.js              ← hash-routed SPA + theme toggle
├── build.js            ← regenerates notes-index.json (Node, no deps)
├── roster.json         ← trainee data (source of truth for the site)
├── notes-index.json    ← list of sessions, regenerated by build.js
├── notes/              ← one markdown file per session
├── .nojekyll           ← tells GitHub Pages to skip Jekyll processing
└── README.md
```

## Routes

The site uses hash routing — works on GitHub Pages with no server config.

- `#/` — Active roster (cards). Tap a card → trainee detail.
- `#/trainee/<id>` — One trainee's progress summary, age, session count.
- `#/sessions` — All archived sessions, most recent first.
- `#/session/<key>` — One session note, parsed from markdown.

## Themes

The site has a sun/moon toggle in the header. It defaults to your system preference, and the choice is remembered in `localStorage` once you tap the toggle.

## Updating content

There are two folders involved:

1. **The source folder** — `wrestling-training/` (one level up from this site folder). The wrestling-training Cowork skill writes to it. It holds the canonical `roster.json` and `notes/`.
2. **This site folder** — `wrestling-training/site/`. This is what gets pushed to GitHub. It needs its own copies of `roster.json` and `notes/`.

After you log a new session or update the roster via Cowork, run:

```bash
cd wrestling-training/site
node build.js --sync
```

That will:
1. Copy `../roster.json` → `roster.json`
2. Copy `../notes/*.md` → `notes/`
3. Regenerate `notes-index.json` from the synced notes

If you'd rather edit `roster.json` and `notes/` directly inside the site folder (skipping the source), just run `node build.js` (no `--sync`) and the index will be rebuilt from whatever's in `notes/` here.

Then commit and push:

```bash
git add .
git commit -m "Update training notes"
git push
```

## Adding the `current_summary` for a new trainee

The trainee detail page reads `current_summary` from `roster.json`. When a new person joins, ask Cowork to write a fresh summary for them — it'll get added to `roster.json` automatically. The same goes for refreshing existing summaries: ask Cowork to "update trainee summaries based on recent sessions" and re-run `build.js --sync`.

## Hosting on GitHub Pages

1. Create a new GitHub repo (public or private — GH Pages works on public free, private with paid).
2. Push the contents of this `site/` folder to the repo root (not the `site/` folder itself — its contents).
3. In the repo, go to **Settings → Pages**, set the source to the `main` branch and `/ (root)`, and save.
4. GitHub will give you the URL (typically `https://<username>.github.io/<repo>/`).

## Local preview

Any static file server works. The simplest:

```bash
cd wrestling-training/site
python3 -m http.server 8000
```

Then open <http://localhost:8000>.

## Notes on the design

- **Typography**: Inter for body text, JetBrains Mono for labels, IDs, dates, and uppercase eyebrows. Both loaded from Google Fonts.
- **Markdown rendering**: handled by `marked` loaded from a CDN (jsDelivr).
- **Theme toggle**: light/dark, with system preference as the default. Choice persists in `localStorage` under the key `tp-theme`.
- **Caching**: data files are fetched with `cache: 'no-cache'` so updates are picked up without hard reloads.
- **Mobile**: layout is single-column on phones, the brand text trims down on very small screens, and tap targets stay above 40px.
- **Accent color**: change `--accent` in `styles.css` to recolor the highlights site-wide; both themes have their own value.
